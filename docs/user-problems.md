# User Problems

## Purpose

이 문서는 Dream Photo가 해결하려는 실제 사용자 문제를 기록한다.

기능은 바뀔 수 있지만,
사용자의 문제는 프로젝트의 중심이 되어야 한다.

---

# Problem 1

## 차번호 사진 찾기

### 상황

사용자는 자신의 차량 번호를 기억하기 위해
번호판 사진을 찍어둔다.

### 문제

몇 개월 후 차량 번호가 필요해진다.

하지만 언제 찍었는지 기억나지 않는다.

사용자는 갤러리를 오랫동안 스크롤하며 찾아야 한다.

### 사용자가 기억하는 방식

- 내 차번호 사진
- 번호판 사진

### 사용자가 기억하지 못하는 것

- 촬영 날짜
- 파일명
- 저장 위치

### Dream Photo가 해결해야 하는 것

사용자가

"차번호"

또는

"번호판"

이라고 검색하면 즉시 찾을 수 있어야 한다.

---

# Problem 2

## 제품 불량 사진 찾기

### 상황

품질 담당자는 불량품을 발견하면
증거 및 이력 관리를 위해 사진을 촬영한다.

### 문제

몇 주 또는 몇 달 후

특정 제품의 불량 사례를 다시 확인해야 한다.

하지만 관련 사진을 찾기가 어렵다.

### 사용자가 기억하는 방식

- 불량품 사진
- 찍힘 불량
- 스크래치 불량
- 특정 제품의 불량 사례

### 사용자가 기억하지 못하는 것

- 촬영 날짜
- 파일명

### Dream Photo가 해결해야 하는 것

사용자가

"불량"

또는

"제품명"

으로 검색하면 관련 사진을 찾아야 한다.

---

# Problem 3

## 여행 사진 찾기

### 상황

사용자는 여행 중 수십~수백 장의 사진을 촬영한다.

### 문제

시간이 지난 후

"강릉 여행 사진"

을 찾고 싶다.

하지만 촬영 날짜를 기억하지 못한다.

### 사용자가 기억하는 방식

- 강릉 여행
- 바다 보러 갔을 때
- 회 먹었던 날

### 사용자가 기억하지 못하는 것

- 정확한 날짜
- 파일명

### Dream Photo가 해결해야 하는 것

사용자가

"강릉"

이라고 검색하면

관련 사진들을 보여줄 수 있어야 한다.

---

# Core Insight

사용자는 사진을 파일로 기억하지 않는다.

사용자는 기억으로 기억한다.

예시:

- 사람
- 장소
- 물건
- 사건

Dream Photo는 이 방식으로 사진을 이해해야 한다.

# Product Hypothesis

사람은 사진을 저장하고 싶은 것이 아니다.

사람은 나중에 기억을 찾고 싶다.

Dream Photo는 사진 저장 앱이 아니라
기억 검색 앱이다.
