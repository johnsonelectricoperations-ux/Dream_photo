# Dream Photo Technical Spike Research

## 목적

Dream Photo의 핵심 기술 가설을 검증한다.

핵심 질문:

사용자는 자연어 검색을 통해
원하는 사진을 실제로 찾을 수 있는가?

---

# Spike 1

## 휴대폰 사진 전체 스캔

목표

Flutter에서 사용자의 사진을 읽을 수 있는지 확인

기술

photo_manager

성공 조건

- Android 지원
- iOS 지원
- 사진 메타데이터 획득

수집 정보

- 사진 ID
- 생성일
- 수정일
- GPS
- 파일 경로

---

# Spike 2

## OCR 성능 검증

목표

사진 속 텍스트를 얼마나 잘 추출하는지 확인

기술

Google ML Kit

테스트 데이터

- 차번호
- 명함
- 영수증
- 제품 라벨

성공 조건

90% 이상 인식

저장 예시

사진

↓

12가3456

↓

검색 가능

---

# Spike 3

## AI 이미지 설명 생성

목표

AI가 사진을 의미 있게 설명할 수 있는지 확인

예시

입력

자동차 사진

출력

"흰색 SUV 차량"

---

입력

불량품 사진

출력

"제품 표면 스크래치"

성공 조건

사용자가 검색 가능한 수준의 설명 생성

---

# Spike 4

## 검색 엔진 검증

목표

사진 검색 속도 측정

기술

SQLite FTS5

테스트 데이터

10,000장

검색 예시

- 강릉
- 차번호
- 불량
- 영수증

성공 조건

1초 이내 검색

---

# Spike 5

## 이벤트 생성

목표

사진을 자동 그룹화 가능 여부 확인

예시

8월 1일~3일

강릉

↓

강릉 여행 이벤트 생성

성공 조건

사용자가 실제 기억 단위로 인식

---

# Spike 6

## 프라이버시 검증

목표

온디바이스 처리 가능성 평가

확인 항목

- OCR
- 검색
- 태그 생성

성공 조건

사진 원본 외부 전송 최소화

---

# MVP Exit Criteria

다음이 가능하면 MVP 개발 시작

- 사진 스캔 가능
- OCR 가능
- 검색 가능
- AI 설명 가능
- 이벤트 생성 가능

---

# Success Definition

사용자가

"차번호"

입력

↓

원하는 사진 발견

또는

"강릉"

입력

↓

강릉 여행 사진 발견

이 경험을 제공할 수 있으면
Dream Photo의 핵심 가설은 검증된 것으로 본다.
